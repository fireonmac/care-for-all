# 제품 기준

## 목표

케어포올은 요양보호사가 그날 관찰한 사실을 짧게 입력하면 공식적인 문체의
일일 인지 기록, 일일 행동 기록, 주간 종합 기록 초안을 만드는 도구다.

핵심 가치는 자동 작성 자체가 아니라 **입력된 사실을 빠뜨리거나 과장하지
않고 검토 가능한 기록 문장으로 바꾸는 것**이다.

## 사용자

주 사용자는 IT 서비스에 익숙하지 않은 요양보호사다. 제품은 사용자가 자유로운
프롬프트 작성법을 익히도록 요구하지 않고, 예측 가능한 선택과 짧은 입력으로
결과를 만들 수 있어야 한다.

## 기록 원칙

다음 원칙은 UI, 프롬프트, 저장 로직보다 우선한다.

- 사용자가 입력하지 않은 사실, 진단, 감정, 원인, 조치를 만들지 않는다.
- 관찰 가능한 행동과 사용자가 입력한 도움·반응만 기록한다.
- 생성 결과는 초안이며, 사용자가 확인하고 수정한 뒤 사용한다.
- 인지와 행동 기록을 구분하되 단순 활동을 문제 행동으로 해석하지 않는다.
- 실명 등 식별정보는 AI 요청에서 가능한 한 분리한다.
- 근거가 부족하면 내용을 채우지 않고 부족함을 알린다.
- 주간 변화나 반복성을 말하려면 둘 이상의 근거가 있어야 한다.

## 문체

- `~합니다`, `~했습니다`, `~보였습니다`, `~드렸습니다` 형식을 사용한다.
- 한 기록은 원칙적으로 1~3문장으로 작성한다.
- 관찰 사실을 먼저 쓰고, 입력된 경우에만 도움과 이후 반응을 쓴다.
- 전문 용어, 가치 판단, 과장된 수식과 개조식 종결을 피한다.

금지 예:

- 근거 없는 진단: `치매가 심해짐`
- 원인 단정: `관심을 받기 위해`
- 가치 판단: `말을 안 들음`
- 입력하지 않은 조치: `정서적 지지를 제공함`
- 근거 없는 변화: `인지 기능이 악화됨`

## 핵심 흐름

1. 대상자를 선택한다.
2. 날짜와 관찰 메모를 입력한다.
3. 일일 인지·행동 기록 초안을 생성한다.
4. 결과를 검토하고 수정하거나 복사한다.
5. 한 주의 일일 기록이 충분하면 주간 종합 기록을 생성한다.

## MVP 범위

포함:

- 대상자 등록과 선택
- 날짜별 관찰 메모 입력
- 일일 인지·행동 기록 생성
- 생성 결과 수정, 복사, 삭제
- 일일 기록을 이용한 주간 기록 생성
- 로컬 Ollama 연동

제외:

- 진단, 위험도 판정, 치료 또는 투약 권고
- 케어포 계정 비밀번호 저장
- 케어포 화면 무인 자동 입력
- 보호자에게 보고서 자동 발송
- 다기관 결제와 복잡한 권한 체계

현재 구현 여부와 다음 작업은 [로드맵](./roadmap.md)에서 관리한다.

## UX 원칙

- 모바일을 우선하고 주요 버튼의 터치 영역과 글자를 충분히 크게 만든다.
- 사용자가 작업 맥락을 잃지 않도록 불필요한 페이지 이동을 줄인다.
- 상태와 결과를 명확한 텍스트로 설명하고 장식보다 기록 작업에 집중한다.
- 색상은 `src/app/globals.css`에 정의된 의미 기반 토큰을 사용한다.
- 접근성이 필요한 복합 UI는 Base UI (headless primitive)를 사용한다.

## 개인정보 기준

대상자의 인지·행동 기록은 민감한 돌봄 정보로 취급한다.

- AI 요청과 로그에 불필요한 식별정보를 넣지 않는다.
- `.env`, 데이터베이스, 백업을 Git에 커밋하지 않는다.
- 운영 접속은 인증된 HTTPS 프록시 또는 사설망으로 제한한다.
- Ollama와 SQLite를 공용 인터넷에 직접 노출하지 않는다.
- 실제 개인정보를 사용하기 전에 인증, 보유 기간, 백업 암호화 기준을 정한다.
