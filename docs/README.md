# 문서 안내

프로젝트의 상세 문서는 주제별로 나뉜다. 같은 내용을 여러 문서에서 설명하지
않고, 아래에 지정된 문서를 해당 주제의 기준으로 사용한다.

| 문서 | 내용 |
| --- | --- |
| [제품 기준](./product.md) | 사용자, 제품 가치, 기록 원칙, UX 원칙, 기능 범위 |
| [아키텍처](./architecture.md) | 기술 스택, 현재 시스템 구조, 코드 경계, 데이터 흐름 |
| [개발 컨벤션](./conventions.md) | 코드 작성 규칙, 모듈 배치, UI·데이터·검증 원칙 |
| [운영 가이드](./operations.md) | 로컬 개발, 검증, 배포, 백업, 복구, 장애 대응 |
| [로드맵](./roadmap.md) | 현재 한계, 미구현 작업, 우선순위와 완료 조건 |

## 갱신 규칙

- 기능의 의도나 기록 기준이 바뀌면 `product.md`를 수정한다.
- 코드 경계나 데이터 흐름이 바뀌면 `architecture.md`를 수정한다.
- 코드 작성 규칙이나 검증 기준이 바뀌면 `conventions.md`를 수정한다.
- 환경변수, 배포, 백업 절차가 바뀌면 원본 설정과 `operations.md`를 함께 수정한다.
- 아직 구현하지 않은 결정은 `roadmap.md`에 두고 완료 후 해당 항목을 제거한다.
- 버전과 스크립트는 문서에 고정값으로 반복하지 않고 `package.json`을 가리킨다.
- 데이터 필드는 문서에 별도 스키마로 재정의하지 않고 `src/db/schema.ts`를 가리킨다.
