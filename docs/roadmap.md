# 로드맵

이 문서는 아직 완료되지 않은 작업만 관리한다. 구현을 마친 항목은 제거하고,
현재 동작 설명이 필요하면 [아키텍처](./architecture.md)로 옮긴다.

## P0: 실제 정보 사용 전

- 애플리케이션 인증과 접근 제어를 추가한다.
- 운영 저장소와 백업의 암호화·보유·삭제 기준을 정한다.
- AI 요청과 로그에서 대상자 식별정보가 분리되는지 검증한다.
- 생성 프롬프트에서 입력에 없는 사실을 보충하도록 하는 지시를 제거한다.
- 생성 결과의 모든 주장에 입력 근거가 있는지 검증하는 단계를 추가한다.

완료 조건: 실제 개인정보를 넣기 전에 접근 통제, 데이터 취급 기준, 사실성
검증이 코드와 운영 절차에 반영되어야 한다.

## P1: 기록 신뢰성

- 원문 관찰 메모와 생성 결과의 근거 연결을 저장한다.
- 생성 결과를 초안/검토/확정 상태로 구분한다.
- 인지·행동 분류를 사용자가 수정할 수 있게 한다.
- 주간 기록은 확정된 관찰 사실을 근거로 생성한다.
- 근거 부족, 금지 표현, 날짜·대상자 혼동을 검사한다.
- 검증 실패 시 보수적 재생성 또는 비 AI 템플릿으로 대체한다.

완료 조건: 문장 단위로 근거를 확인할 수 있고, 입력에 없는 사실이 최종
기록에 들어가지 않는 회귀 테스트가 있어야 한다.

## P2: 사용성

- 사건, 관찰, 제공한 도움, 이후 반응을 짧은 단계로 입력하게 한다.
- 자주 쓰는 사건 선택지를 제공한다.
- 모호한 입력에는 자유 질문 대신 선택형 확인 질문을 최대 한 개 제시한다.
- 기록 상태를 `미작성`, `메모 있음`, `검토 필요`, `완료`로 구분한다.
- 모바일 큰 글씨와 고대비 사용성을 검증한다.
- 네트워크 장애 중 임시 입력 보존 방식을 정한다.

## P3: 품질 평가

- 식별정보를 제거한 관찰 메모와 교정 문장으로 골든 데이터셋을 만든다.
- 사실 추가, 대상자·날짜 혼동, 금지 표현, 영역 분류를 자동 평가한다.
- 모델과 프롬프트 버전을 생성 결과에 남긴다.
- 실제 사용자의 수정량과 생성 소요 시간을 측정한다.

목표 기준:

- 사실 추가 0건
- 대상자·날짜 혼동 0건
- 금지 표현 0건
- 인지·행동 분류 정확도 95% 이상

## 보류

- 음성 받아쓰기
- PostgreSQL 전환과 다기관 지원
- 공식 API가 확인된 경우의 케어포 연동
- Apple Silicon 최적화를 위한 MLX-LM 비교

케어포 화면 좌표를 클릭하는 무인 자동화는 UI 변경, 오입력, 계정 보안 위험
때문에 제품의 기본 방향으로 채택하지 않는다.
