# 주야간보호 업무 일지 생성 서비스 - 개발 계획서 (Development Plan)

본 계획서는 최소한으로 요구되는 핵심 MVP 기능부터 단계적으로 빌드업하기 위한 구체적인 개발 로드맵입니다.

---

## 단계 1: 프로젝트 초기 설정 및 기반 구축 (Phase 1)
**목표:** 프론트엔드 환경 및 로컬 DB를 세팅합니다.

1. **Next.js 프로젝트 생성**
   - 패키지 매니저로 `pnpm`을 사용하여 Next.js 15 (App Router), TypeScript, Tailwind CSS 세팅 (`pnpm create next-app`).
2. **디자인 토큰 설정 (`tailwind.config.ts`)**
   - 시맨틱 색상 설정 및 친근한 느낌을 주는 UI 라운드 처리(border-radius).
3. **Headless UI 준비**
   - Base UI 패키지 설치.
4. **로컬 DB 설정**
   - SQLite 및 Drizzle ORM 연동 설정. `Recipient` (대상자) 및 `Record` (기록) 스키마 정의.

---

## 단계 2: 메인 화면 및 대상자 등록 (Phase 2)
**목표:** 대상자 리스트를 보고 새로운 대상자를 등록하는 기능을 구현합니다.

1. **메인 화면 (대상자 리스트)**
   - 등록된 대상자 목록을 불러와 리스트 형태로 화면에 렌더링.
2. **대상자 등록 기능**
   - '대상자 등록' 버튼 클릭 시 나타나는 입력 창 구현.
   - 이름 입력 후 확인 버튼을 누르면 DB에 저장하고 리스트를 즉각 갱신.

---

## 단계 3: 대상자 상세 화면 (주간 내역 및 당일 입력) (Phase 3)
**목표:** 대상자의 기록을 조회하고 당일 관찰 키워드를 입력받습니다.

1. **주간 입력 내역 뷰**
   - 대상자 클릭 시 상세 화면으로 이동.
   - 현재 주(Week)에 입력된 과거 기록 내역 리스트업.
2. **오늘의 입력 내역 UI**
   - 당일 기록이 비어있을 경우 명시적으로 입력을 유도하는 UI 표시.
   - 관찰된 내용을 키워드 형태(텍스트/칩)로 여러 개 입력할 수 있는 폼 구성.

---

## 단계 4: 일일 요양보호기록 자동 포맷팅 및 저장 (Phase 4)
**목표:** 요양보호사가 대충 입력한 단편적인 키워드/문장들을 공식적인 요양보호기록 포맷(인지/행동)으로 변환해주는 서비스의 핵심 가치를 구현합니다.

1. **AI 변환(포맷팅) 프롬프트 연동**
   - 사용자는 문장 완성에 대한 부담 없이 단편적인 문장이나 여러 키워드만을 자유롭게 입력함.
   - AI는 전달받은 파편화된 정보들을 분석하여, 실제 기관에서 요구하는 공식적인 포맷의 **"인지 관련 요양보호 문장"**과 **"행동 관련 요양보호 문장"**으로 명확히 구분하여 문맥에 맞게 포맷팅해 줌.
2. **생성 및 저장**
   - 생성된 인지/행동 기록 문장을 화면에 표시.
   - 확인 후 DB에 당일 기록으로 최종 저장.

---

## 단계 5: 주간 리포트 발간 기능 (Phase 5)
**목표:** 최소 2일 이상의 기록이 모였을 때 주간 종합 기록을 생성합니다.

1. **리포트 발간 버튼 활성화 조건 로직**
   - 해당 대상자의 이번 주 입력 기록이 **2일 이상** 존재하는지 검사.
   - 조건 만족 시 '주간 리포트 발간' 버튼을 화면에 노출.
2. **주간 기록 종합 및 생성**
   - 버튼 클릭 시 해당 주간의 모든 일일 기록(인지/행동) 데이터를 AI로 전달.
   - AI가 해당 데이터를 바탕으로 **인지, 행동이 종합된 한 주간의 요양보호기록**을 작성.
3. **발간 내용 저장**
   - 생성된 주간 리포트 내용을 확인하고 DB에 최종 발간 내용으로 저장.
